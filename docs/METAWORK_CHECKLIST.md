# ops-dashboard Meta-work Checklist

Use this checklist during active development to keep project docs current.

## Update when structure changes
- [ ] navigation/module structure changed
- [ ] page/module names changed
- [ ] boundary between modules changed
- [ ] control-center flow changed

## Update when data flow changes
- [ ] new endpoint added
- [ ] endpoint fields changed
- [ ] payload ownership moved between modules
- [ ] caching/reuse behavior changed
- [ ] loading/progress behavior changed

## Update when UX rules change
- [ ] table sorting/search/filter rules changed
- [ ] dictionary mapping rules changed
- [ ] load gating or selection behavior changed
- [ ] explanation boundary changed

## Docs to touch
- `PROJECT_STATUS.md`
- `PROJECT_MAP.md`
- `DATA_CONTRACTS.md`
- `WORKSTREAM_DECISIONS.md`
- `DASHBOARD_DESIGN.md`

## Rule
Do this during the work, not after the work.
