# trading-dashboard

Website dashboard repository for the trading system.

## 1. Repository role

`trading-dashboard` is the downstream dashboard/UI repository for the trading stack.

It is responsible for:
- website dashboard construction
- dashboard UI/page/module design
- local server / adapter layer for dashboard use
- dashboard-facing data contracts
- shared dictionary/display metadata used by the dashboard
- visualization of outputs already generated by the trading repos

It is **not** responsible for:
- upstream market-data acquisition
- strategy replay computation
- model-training or ranking internals
- live execution internals
- cross-repo scheduling/timing policy
- centralized cross-repo report generation

## 2. System boundary

Current rule:
- each trading repo keeps its own output/report generation responsibility
- `trading-dashboard` only visualizes those outputs in the website dashboard

## 3. Current product structure

The dashboard should stay modular.

Current major product surfaces:
1. Welcome
2. Historical Backtest
3. Trading Performance
4. Market State Analysis

Boundary rule:
- Historical Backtest = control center / pre-load workspace
- Trading Performance = trading-and-return analysis
- Market State Analysis = state/cluster interpretation

## 4. Run

```bash
python3 server.py
```

Default local server:
- <http://localhost:8787>

- `index.html` — main front-end UI
